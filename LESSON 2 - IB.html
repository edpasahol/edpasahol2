Reflection and Example

Fourth Quarter Lesson

Reflection

In this lesson, I learned how the placement of JavaScript affects its execution. Scripts in the <head> run before the page content loads, suitable for function definitions. Scripts in the <body> run as the page content loads, useful for immediate outputs. Using external JavaScript files enhances code reuse, simplifies maintenance, and avoids redundancy. Comments, whether single-line (//) or multi-line (/* ... */), improve readability and debugging. I also learned important coding rules: JavaScript is case-sensitive, whitespace is flexible for readability, and line-breaking must be done carefully to avoid syntax errors. Mastery of these practices is essential for creating clean, functional, and maintainable web scripts.

Example JavaScript Code

Hover over a code box to see its explanation.

<head>
<script>
  function greet() {
    alert("Hello Visitor!");
  }
</script>
</head>
<body>
<script>
  document.write("This message is displayed as the body loads.");
</script>
</body>
<!-- External JS File -->
<script src="myScript.js"></script>
<!-- Note: myScript.js contains only JS code, no <script> tags inside -->

Browser Output

Alerts from head scripts appear first.
Text from body scripts appears as the page content loads.
External JS behaves like any other script, allowing modular code reuse.
